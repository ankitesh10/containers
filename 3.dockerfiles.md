# Dockerfiles

## Create Dockerfile

Sample code:

```dockerfile
FROM node:20

CMD ["node", "-e", "console.log(\"hi lol\")"]
```

`CMD` runs this file during execution with `docker run`, not during build.

## Build Container

```sh
docker build .
```

## Run Container

```sh
docker run <hash_id>
```

Example hash:

```text
bdb6f833942381b166027b3c14d099030e5152c71a0d10883ee4906821789807
```

## Tag Images

`-t` adds a tag.

```sh
docker build . -t my-node-app
docker run my-node-app
docker run --init my-node-app
```

`--init` allows exiting via `Ctrl+C`, but it will not do it gracefully or allow the process to complete.

Versioning:

```sh
docker build . -t my-node-app:2
```

## Expose Port

The first port is the host machine port, and the second port is the container port.

```sh
docker run --init --publish 3000:3000 --rm --name na my-node-app
```

There is a Dockerfile instruction called `EXPOSE 3000`, which suggests a port to expose.

There is another flag for respecting those exposed ports, which randomly assigns an exposed port and is not convenient. So `--publish` is generally expected standard.

## Creating A Server

```dockerfile
FROM node:20

COPY index.js index.js

CMD ["node", "index.js"]
```

## Define Proper Owner And Path

```dockerfile
FROM node:20

COPY index.js index.js

CMD ["node", "/home/node/code/index.js"]
```

## Define User

You can define user `node` in the instruction. By default, it will be `root`.

`node` is a default user created by the Node Docker image.

Create a new user:

```dockerfile
RUN useradd -ms /bin/bash lolcat
```

Example:

```dockerfile
FROM node:20

USER node

COPY index.js /home/node/code/index.js

CMD ["node", "/home/node/code/index.js"]
```

## Enter Docker Bash

```sh
docker exec -it na bash
```

When copying, add `--chown`. This is redundant if user `node` is added.

```dockerfile
COPY --chown=node index.js /home/node/code/index.js
```

Instead of `COPY`, you can also use `ADD`, which has much more permissions.

## Workdir

```dockerfile
FROM node:20

USER node

WORKDIR /home/node/code

COPY index.js index.js

CMD ["node", "index.js"]
```

## Complicated JS App

```dockerfile
FROM node:20

USER node

WORKDIR /home/node/code

COPY --chown=node:node . .

RUN npm ci

CMD ["node", "index.js"]
```

## Layers

```dockerfile
FROM node:20

USER node

WORKDIR /home/node/code

COPY --chown=node:node package*.json ./

RUN npm ci

COPY --chown=node:node . .

CMD ["node", "index.js"]
```

The Dockerfile above installs packages only when `package.json` and `package-lock.json` change. Otherwise, it takes dependencies from cache.
