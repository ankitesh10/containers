# Docker Compose

Reference repo for code and sample Docker Compose file:

- <https://github.com/btholt/complete-intro-to-containers-v2>

With Docker Compose, we simplify this a lot. Docker Compose allows us the ability to coordinate multiple containers and do so with one YAML file.

Docker does say that Docker Compose is suitable for production environments if you have a single instance running multiple containers.

In addition to working very well in development, Docker Compose is very useful in CI/CD scenarios when you want GitHub Actions or some CI/CD provider to spin up multiple environments to quickly run some tests.

## Compose File

Create a new file in the root directory of your project called `docker-compose.yml` and put this in there:

```yaml
services:
  api:
    build: api
    ports:
      - "8080:8080"
    links:
      - db
    environment:
      MONGO_CONNECTION_STRING: mongodb://db:27017
  db:
    image: mongo:7
  web:
    build: web
    ports:
      - "8081:80"
```

Reference:

- <https://docs.docker.com/reference/compose-file/#compose-file-structure-and-examples>

## Run

```sh
docker compose up
```

If you change something and want to make sure it builds, run:

```sh
docker compose up --build
```

Docker Compose is not watching for changes when you run `up`.

## Nodemon

```dockerfile
FROM node:latest

RUN npm i -g nodemon

USER node

RUN mkdir /home/node/code

WORKDIR /home/node/code

COPY --chown=node:node package-lock.json package.json ./

RUN npm ci

COPY --chown=node:node . .

CMD ["nodemon", "index.js"]
```

Now we can write our code and every time it saves, it will restart the server from within the container. This will make this productive to work with.

While Kubernetes handles bigger deployment scenarios than Docker Compose can, you can use this command to scale up your `web` container to 10 concurrently running containers:

```sh
docker compose up --scale web=10
```
